# RCA Copilot — 快速上手

> 製造現場出現異常（良率下降、參數偏移、訊號異常）時，這個工具幫你**更快整理可能原因、佐證線索，以及下一步該檢查什麼**。
>
> 🧭 **它是 Copilot，不是 Judge。** 它給的是「候選根因」，不是定論；真正原因仍要你在現場依設備、製程與維修紀錄確認。

---

## 1. 啟動

```bash
./start.sh
```
瀏覽器開 **http://localhost:8724**。（後端 API 在 8723，需 `claude` CLI 已登入。）

---

## 2. 三分鐘跑第一次分析

1. **新建專案**：左側欄 `＋` 開一個新 thread（例如「Line-3 良率下降」）。
2. **選 RCA 模式**：在上方 `WORKFLOW` 下拉選 **單代理 RCA**（最快）。畫面會切成 RCA 工作區，顯示步驟：`Anomaly Intake → RCA Analysis`。
3. **描述異常**：點 **Anomaly Intake**，把症狀寫清楚（什麼指標異常、何時開始、哪條線/機台/產品）。按 **Generate** 讓它整理成 intake brief。
4. **上傳資料 + 分析**：點 **RCA Analysis**，把製程/良率資料（CSV、Excel、PDF…）拖上來，按 **Generate**。
5. **看結果**：你會得到一張**候選根因表** —— 每個候選附「信心程度／證據（引用你資料裡的具體數據）／建議的下一步檢查」，並排出建議檢查順序。每列可按 **✓ / ✕ / ?** 標記你的裁決（輔助用）。
6. **到現場確認**：挑信心高、成本低的候選先查。工具只是幫你縮短「該查什麼」的時間。

> ⚠️ **資料要掛在會讀它的步驟**：單代理模式請把資料上傳在 **RCA Analysis**；多代理鏈模式請上傳在 **Baseline** 步驟。（每個步驟只讀掛在它自己身上的附件。）

---

## 3. 三種模式，怎麼選？

| 模式 | 適合 | 它會做什麼 |
|---|---|---|
| **單代理 RCA** | 快速初判 | 一個助手直接讀資料、給候選根因表 |
| **多代理鏈** | 要看推理過程 | 分四步：量化偏移 → 因果圖推理 → 對照已知失效模式 → 彙整（因果步驟會畫**因果圖**）|
| **Agentic 規劃** | 不確定該跑哪些步驟 | 讓 AI 先**提案一份分析計畫**（要跑哪些步驟），你**核准**後它才依計畫執行 |

每個步驟都能 **Refine**（用一句話要它補強，例如「第 1 個候選請給可量化的判定門檻」）和 **Approve**（核准後才往下一步）。改了上游，下游會自動標成「需重做」。

**Agentic 規劃流程**：選 `Agentic 規劃` → 在 Intake 描述異常 → 到 **RCA Plan** 按 Generate（AI 產出計畫）→ 檢視後按 **「核准並套用 plan」** → 工具自動建立對應流程並切過去，逐步執行。

## 4. 想先試玩？載入範例情境（可選）

附三個合成（假資料）情境，一鍵載入即可體驗：

```bash
cd backend && python -m scripts.seed_rca
```
- **RCA：Line-3 良率下降**（單代理）— 良率步階下降
- **RCA：ETCH 參數漂移**（多代理鏈）— 機台參數慢慢跑出規格
- **RCA：製程訊號間歇異常**（多代理鏈）— 感測訊號間歇異常

載入後在左側欄就能看到這三個專案，直接點 Generate 試跑。

**清掉範例 / 還原乾淨**：
```bash
cd backend && python -m scripts.reset_rca_demo
```
（只會刪這三個範例 + 規劃示範，不動你自己的專案。）

---

## 5. 幾個重點

- **輸出是候選、不是判決**：每份結果都會標明信心與證據，結尾提醒「待工程師確認」。請當成「縮短排查時間」的助手。
- **證據可追溯**：候選根因會引用你資料中的具體列/欄/趨勢，方便你回頭驗證。
- **資料越具體越好**：附上良率/參數/訊號的原始檔（含機台、時間、批號欄位），分析品質會明顯提升。
- **目前是 PoC**：知識/SOP 比對目前用「一般製造領域知識」；接上你廠的 MES/FDC/SOP 知識庫後，候選會更貼近你現場。

更技術性的架構與檔案說明見 [`docs/RCA.md`](./RCA.md)。
